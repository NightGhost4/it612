setTimeout with 0ms delay still runs last

The Puzzle

  • setTimeout(fn, 0) defers, not skips - Even with 0ms delay, the callback is queued, not executed immediately
  • Sync code always finishes first - All synchronous code in the main thread completes before async tasks run
  • A 20ms timer can fire at 55ms - If synchronous code blocks the thread, timers are delayed
let start = Date.now();
setTimeout(() => {
  console.log("Timeout ran at", Date.now() - start);
}, 20);
while (Date.now() < start + 50) {}
console.log("Wasted time until", Date.now() - start);
// -> Wasted time until 50
// -> Timeout ran at 55
